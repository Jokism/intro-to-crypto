Overview of Cryptography

What is Cryptography?

  • Let's pretend we have two protagonists: Alice and Bob.
  • Alice wants to send Bob a message but she has to pass the note through an unsafe medium
  • The aim of cryptography is to provide Alice with the primitives (aka tools) to encrypt her messages such that: no one can eavesdrop on her communication with Bob and no one can tamper with the communication.
  • Definition: secure communication through untrusted mediums

What are Cryptography's applications?

  • Digital Signatures
  • Anonymous Communication
  • Multiparty computation
  • Anonymous Cash (not cryptocurrencies)
  • Private computation on untrusted hardware.
  • Zero-knowledge systems
  • And many more...

History of Cryptography

Heiroglyphics (~1900 BC)

  • Kings and their scribes would write messages in Heiroglyphics which was only known to them.

Flaws in Heiroglyphics?

  • Anyone who learned this "code" could read them plain as day.
  • Modern day equivalent to writing a message in Arabic and just hoping that no one can read it.

Caesar Cipher (~100 BC)

  • Used by Caesar to communicate to his generals
  • shift numbers over by 3
					  alphabet = ["a", "b", "c", ... , "z"]
secret = "hello"
shift = 3

def encode_message(plain_secret, shift):
	encrypted_message = ""
	for i in range(len(plain_secret)):
		curr_index = alphabet.index(plain_secret[i])
		encrypted_message += alphabet[(shift + curr_index) % 26]
	return encrypted_message

def decode_message(encrypted_secret, shift):
	plain_message = ""
	for i in range(len(encrypted_secret)):
		curr_index = alphabet.index(encrypted_secret[i])
		plain_message += alphabet[(curr_index - shift) % 26]
	return plain_message

Flaws in Caesar?

  • only 26 keys, easy to brute force
  • A common flaw is that heiroglyphics rely on secrecy of the system, not necessarily the keys

Vigenere Cipher (~1500 AD)

  • First to use a key
  • Essentially Caesars but with a longer key
  • Key must be less than the length of the message 
					alphabet = ["a", "b", "c", ... , "z"]
key = "abc"
secret = "hello"

def encrypt(plain_secret, key):
	encrypted_secret = ""
	shifts = [alphabet.index(key[i % len(key)]) for i in range(len(plain_secret))]
	for i in range(len(plain_secret)):
		shift = shifts[i]
		curr_index = alphabet.index(plain_secret[i])
		encrypted_secret += alphabet[(curr_index + shift) % 26]
	return encrypted_secret

def decode(encrypted_secret, key):
	plain_secret = ""
	shifts = [alphabet.index(key[i % len(key)]) for i in range(len(encrypted_secret))]
	for i in range(len(encrypted_secret)):
		shift = shifts[i]
		curr_index = alphabet.index(encrypted_secret[i])
		plain_secret += alphabet[(curr_index - shift) % 26]
	return plain_secret

Flaws in Vigenere?

  • If we knew the key length, we can break using something known as frequency analysis
  • If we didn't know the key length, we can break by trying all key lengths

Enigma Machine

  • Similar to other substitution cyphers, except using multiple rotating rotors.
  • Vulnerable to frequency analysis as well

Modern Cryptography

One Time Pads

OTP Mechanism

  • Like a Vigenere cipher with a key as long as the message. 
  • Key = {0, 1}n
Message = {0, 1}n 
EncryptedMessage = {0, 1}n
Message = Key XOR EncryptedMessage
EncryptedMessage = Key XOR Message

OTP Security

  • Given a ciphertext, an adverserial attacker will learn nothing about the original message.
  • Early cryptography relied on intuition, but intuition is incomplete.
  • Lets translate this definition into something more mathematical
  • Given a cipher text C, the probably that it is the encryption of any two messages M is equally likely

OTP Weaknesses

  • Can only be used once. (if you uses the same pad on two messages and xor those two messages together, you learn the xor of those two messages)
  • No integrity! If you remember from before, it is important that we can demonstrate messages were not tampered with.
  • The way you can break integrity is just xoring the encrypted text with another random key.
